Per-world HUD & unified buttons

The two highest-leverage fixes, in-language: a HUD that re-themes between the daytime and candy-night worlds so the level title always reads, and a matched pair of control buttons. Before / after for each, with the exact CSS to drop in.

01

HUD that re-themes per world

Same component, two themes driven off WORLDS[n].mood. Daytime keeps ink-purple text on a brighter glass bar; candy-night switches to white text with an ink shadow and a darker translucent fill so the title stops sinking into the sky.

The theming hook

// one class on the HUD, set from the active world's mood
hud.className = 'hud ' + (world.mood === 'candy-night' ? 'night' : 'day');

/* CSS */
.hud.day   { background:rgba(255,255,255,.78); }
.hud.day   .title,.val { color:#34215a; }            // ink
.hud.night { background:rgba(52,33,90,.42); }
.hud.night .title,.val { color:#fff;
             text-shadow:0 2px 0 #1d1336; }     // white + ink shadow
02

One button family

Right now pause is a white circle and mute is a dark circle — they read as two systems. Unify on the white-fill + ink-outline + hard-shadow treatment the rest of the game uses. The active/muted state can invert to ink-fill, so “dark” still means something — it signals state, not a different button.

BEFORE — mismatched
AFTER — unified (sound on)
AFTER — muted state (inverts)
  • Same silhouette: both 48px, 4px ink border, 0 5px 0 #34215a shadow.
  • Default = white fill, ink glyph. Use for pause and sound-on.
  • State change = invert to ink fill, white glyph — and swap the glyph to a clear ✕ over the speaker for “muted”. Dark now reads as “off”, which is meaningful, instead of an arbitrary style difference.
